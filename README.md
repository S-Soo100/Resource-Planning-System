# KARS (Kangsters Auto Resource-management System) - 다양한 품목/창고를 가진 기업을 대상으로 하는 자동화 재고관리 재고 관리 시스템

**버전: v1.5.0**

## 프로젝트 개요

KARS는 다양한 품목/창고를 가진 기업을 대상으로 하는 자동화 재고관리를 위한 통합 재고 관리 시스템입니다. 이 시스템은 재고 관리, 발주 관리, 패키지 관리, 시연 관리 등 운영에 필요한 핵심 기능을 제공합니다.

## 주요 기능

### 🔐 권한 관리 시스템

- **4단계 권한 체계**: Admin, Moderator, User, Supplier
- **권한별 기능 제한**: 각 권한 레벨에 맞는 기능 접근 제어
- **팀 기반 데이터 격리**: 팀별 독립적인 데이터 관리

### 📦 재고 관리

- **실시간 재고 조회**: 창고별, 카테고리별 재고 현황
- **입출고 관리**: 입고/출고 작업 및 이력 추적
- **재고 알림**: 재고 부족 시 자동 알림
- **재고 수정**: 권한에 따른 재고 수량 직접 수정

### 🛒 발주 시스템

- **다양한 발주 유형**: 개별 품목, 패키지, 휠체어 전용 발주
- **승인 워크플로우**: 권한별 단계적 승인 프로세스
- **실시간 상태 추적**: 발주 진행 상황 실시간 모니터링
- **댓글 시스템**: 발주별 소통 및 협업을 위한 댓글 기능
- **파일 첨부**: 발주 관련 문서 업로드 지원

### 🎯 시연 시스템 (v1.5.0)

- **제품 시연 전용**: 일반 발주와 구분된 시연 관리
- **자동 재고 연동**: 시연 출고/복귀 시 자동 재고 반영
- **시연 전용 상태**: 시연 프로세스에 최적화된 상태 관리

### 🦽 휠체어 발주 시스템 (v1.5.0)

- **전용 창고 제한**: 휠체어 전용 창고로만 발주 가능
- **카테고리 필터링**: 휠체어 관련 품목만 선택 가능
- **보라색 테마**: 일반 발주와 구분되는 시각적 디자인

### 💬 댓글 시스템 (v1.5.0)

- **실시간 소통**: 발주별 즉시 피드백 및 질문
- **권한 기반 관리**: 작성자 본인 수정/삭제, Admin 전체 관리
- **사용자 친화적 UI**: 직관적인 댓글 작성 및 관리
- **시간 표시**: 상대적 시간 표시 (방금 전, N분 전 등)
- **수정 표시**: 수정된 댓글의 경우 "(수정됨)" 표시

### 📊 관리 기능

- **사용자 관리**: 팀별 사용자 추가/수정/삭제
- **창고 관리**: 창고 정보 및 권한 설정
- **카테고리 관리**: 품목 카테고리 체계 관리
- **공급업체 관리**: 거래처 정보 관리

### 🔧 공통 컴포넌트 시스템 (v1.5.0)

- **재사용 가능한 컴포넌트**: 주소 검색, 파일 업로드, 담당자 정보 등
- **테마별 색상**: 파란색(일반), 보라색(휠체어) 테마 지원
- **반응형 디자인**: 모바일 환경 최적화
- **접근성**: 모든 입력 필드에 적절한 라벨 및 접근성 속성

## 기술 스택

- **프론트엔드**: React, TypeScript, Tailwind CSS
- **백엔드**: Node.js, Express
- **데이터베이스**: MongoDB
- **인증**: JWT (JSON Web Tokens)
- **상태 관리**: Zustand
- **UI 컴포넌트**: Headless UI, Heroicons
- **폼 관리**: React Hook Form
- **API 통신**: Axios
- **날짜 처리**: date-fns
- **파일 업로드**: AWS S3

## 시스템 요구사항

- Node.js 18.0.0 이상
- MongoDB 4.0 이상
- 최신 버전의 웹 브라우저 (Chrome, Firefox, Safari, Edge)

## 설치 및 실행 방법

1. 저장소 클론

```bash
git clone https://github.com/yourusername/kars.git
cd kars
```

2. 의존성 설치

```bash
npm install
```

3. 환경 변수 설정

```bash
cp .env.example .env
# .env 파일을 열어 필요한 환경 변수 설정
```

4. 개발 서버 실행

```bash
npm run dev
```

## 창고 접근 권한 관리 가이드 (v1.2.0)

### 관리자용 - 사용자 창고 권한 설정

1. **사용자 생성 시 권한 설정**

   - 관리자 페이지 → 팀 멤버 → "새 사용자 생성" 클릭
   - 기본 정보 입력 후 "창고 접근 제한" 섹션에서 제한할 창고 선택
   - 체크된 창고는 해당 사용자가 접근할 수 없습니다

2. **권한 확인**
   - 생성된 사용자는 제한된 창고에 대해 다음과 같은 제한이 적용됩니다:
     - 재고 조회 시 제한된 창고의 데이터 숨김
     - 발주 요청 시 제한된 창고 선택 불가
     - 발주 기록에서 제한된 창고의 발주 기록 숨김

### 사용자용 - 권한 제한 이해

- **접근 가능한 창고**: 정상적으로 모든 기능 사용 가능
- **접근 제한된 창고**:
  - 창고 목록에서 "접근 불가" 표시되거나 숨김
  - 발주 요청 시 선택 시도하면 경고 메시지 표시
  - 관련 데이터가 화면에 표시되지 않음

### 권한 레벨별 특징

| 권한 레벨 | 창고 제한 적용 | 관리 기능 | 발주 승인 | 특이사항            |
| --------- | -------------- | --------- | --------- | ------------------- |
| Admin     | ❌             | 전체      | ✅        | 모든 창고 접근 가능 |
| Moderator | ✅             | 읽기 전용 | ✅        | 재고 수정 불가      |
| User      | ✅             | ❌        | ❌        | 기본 사용자         |
| Supplier  | ✅             | ❌        | ❌        | 발주 메뉴만 접근    |

## 사용자 역할 및 권한

### 1. 관리자 (Admin)

- 모든 기능에 대한 완전한 접근 권한
- 모든 창고에 대한 접근 권한 (restrictedWhs 제한 없음)
- 사용자 계정 관리 및 창고 접근 권한 설정
- 시스템 설정 관리
- 재고 및 발주 관리
- 보고서 생성 및 분석

### 2. 1차승인권자 (Moderator)

- 읽기 전용 모드로 관리 기능 접근
- 발주 승인 권한
- 창고 접근 권한 제한 적용 (`restrictedWhs`)
- 재고 조회 (수정 불가)
- 발주 기록 확인

### 3. 일반 사용자 (User)

- 재고 조회
- 발주 요청
- 발주 기록 확인
- 창고 접근 권한 제한 적용 (`restrictedWhs`)
- 제한된 보고서 접근

### 4. 외부업체 (Supplier)

- 발주 관련 기능만 제한적 접근
- 발주 메뉴만 표시
- 창고 접근 권한 제한 적용 (`restrictedWhs`)
- 발주 상태 변경 불가
