## 잠정 결정 사항

### 1. 재구매 예정 고객 표시 위치
- **결정**: 별도 목록 페이지 + 메인에서 접근 가능한 진입점
- **이유**: 매일 확인해야 하는 운영 정보이므로 쉽게 접근 가능해야 함
- **구현**: `/customer/repurchase` 또는 기존 메뉴 내 탭으로 구현
- **재확인 필요**: 메인 대시보드 위젯 필요 여부

### 2. 고객 서류 접근 위치
- **결정**: 발주 상세 페이지에 서류 섹션 추가
- **이유**: 고객 서류는 주로 발주 맥락에서 확인/관리
- **구현**: 발주 상세 `[id]/page.tsx`에 고객 서류 탭/섹션 추가
- **재확인 필요**: 고객 목록 페이지에서도 서류 접근 필요한지

### 3. 출고완료 건 수정 UI
- **결정**: 기존 DetailsEditModal 확장
- **이유**: 이미 supplierId, 가격 수정이 구현되어 있으므로 환급/입금/세금계산서 필드 추가
- **구현**: DetailsEditModal에 섹션별로 폼 필드 추가

### 4. 주민등록번호 마스킹
- **결정**: 입력 후 표시 시 `900101-*******` 형태로 마스킹
- **이유**: 개인정보 보호
- **구현**: 입력은 평문, 표시는 마스킹, 저장은 암호화된 값 그대로 전송

### 5. 세금계산서 표시 위치
- **결정**: 발주 상세 페이지에 세금계산서 섹션 추가
- **이유**: 세금계산서는 주문(Order)에 종속된 데이터
- **구현**: 파일 업로드 + 목록 + 삭제 기능

### 6. 입금 상태 UI
- **결정**: Select 드롭다운
- **이유**: 고정 옵션 5개 (자부담금/전액/선금/중도금/잔금)이므로 드롭다운이 적합
- **구현**: Ant Design Select 컴포넌트
